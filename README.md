# Пошук у ширину (Breadth-First Search)

## 1. Основи графів: вершини, ребра, шляхи

**Граф** — це структура даних, яка складається з:

- **Вершин (vertices / nodes)** — точок, що представляють об'єкти (наприклад, міста, людей, веб-сторінки).
- **Ребер (edges)** — зв'язків між вершинами (наприклад, дорога між містами, дружба між людьми, посилання між сторінками).

Типи графів:

| Тип | Опис |
|---|---|
| Орієнтований (directed) | Ребра мають напрямок: A → B не означає B → A |
| Неорієнтований (undirected) | Ребра двосторонні: якщо A–B, то й B–A |
| Зважений (weighted) | Кожне ребро має "вагу" (вартість, відстань, час) |
| Незважений (unweighted) | Усі ребра рівнозначні |

**Шлях (path)** — послідовність вершин, з'єднаних ребрами, що веде від однієї вершини до іншої.

**Сусіди (neighbors)** — вершини, безпосередньо з'єднані ребром з даною вершиною.

## 2. Представлення графів у пам'яті

### Список суміжності (adjacency list)
Найпоширеніший спосіб. Кожній вершині відповідає список її сусідів.

```python
graph = {
    "A": ["B", "C"],
    "B": ["A", "D"],
    "C": ["A"],
    "D": ["B"],
}
```

**Переваги:** економить пам'ять для розріджених графів (де мало ребер), швидкий доступ до сусідів вершини.

### Матриця суміжності (adjacency matrix)
Двовимірна таблиця, де `matrix[i][j] = 1`, якщо є ребро між вершиною `i` та `j`, інакше `0`.

```
    A  B  C  D
A [ 0  1  1  0 ]
B [ 1  0  0  1 ]
C [ 1  0  0  0 ]
D [ 0  1  0  0 ]
```

**Переваги:** швидка перевірка наявності ребра (O(1)).
**Недоліки:** багато пам'яті для розріджених графів (O(V²)).

Для BFS зазвичай використовують **список суміжності**, оскільки він ефективніший при переборі сусідів.

## 3. BFS: алгоритм і його реалізація

### Ідея алгоритму

BFS досліджує граф "шарами" (рівнями), починаючи з заданої вершини:

1. Спочатку відвідуємо стартову вершину.
2. Потім — усіх її прямих сусідів (рівень 1).
3. Потім — сусідів цих сусідів, яких ще не відвідали (рівень 2).
4. І так далі, поки не залишиться невідвіданих досяжних вершин.

Це гарантує, що BFS знаходить **найкоротший шлях** між вершинами у незваженому графі (мінімальну кількість ребер).

### Структура даних: черга (queue)

BFS використовує **черга (FIFO — First In, First Out)**: вершина, додана першою, обробляється першою. Це і забезпечує обхід "шар за шаром".

### Псевдокод

```
BFS(graph, start):
    visited = {start}
    queue = [start]

    while queue не порожня:
        vertex = queue.pop_front()
        обробити(vertex)

        для кожного neighbor у graph[vertex]:
            якщо neighbor не у visited:
                visited.add(neighbor)
                queue.push_back(neighbor)
```

### Реалізація на Python

```python
from collections import deque

def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    order = []

    while queue:
        vertex = queue.popleft()
        order.append(vertex)

        for neighbor in graph.get(vertex, []):
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

    return order
```

### Складність алгоритму

- **Час:** O(V + E) — кожна вершина і кожне ребро обробляються один раз.
- **Пам'ять:** O(V) — для черги та множини відвіданих вершин.

### BFS для пошуку найкоротшого шляху

Окрім простого обходу, BFS можна модифікувати так, щоб відстежувати шлях до кожної вершини — це дає найкоротший шлях у незваженому графі:

```python
def bfs_shortest_path(graph, start, goal):
    if start == goal:
        return [start]

    visited = {start}
    queue = deque([[start]])

    while queue:
        path = queue.popleft()
        vertex = path[-1]

        for neighbor in graph.get(vertex, []):
            if neighbor == goal:
                return path + [neighbor]
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(path + [neighbor])

    return None  # шляху не існує
```

### Підрахунок рівнів (відстаней) за допомогою BFS

BFS також можна використати, щоб дізнатися відстань (у кількості кроків) від стартової вершини до кожної іншої вершини графа:

```python
def bfs_levels(graph, start):
    distances = {start: 0}
    queue = deque([start])

    while queue:
        vertex = queue.popleft()

        for neighbor in graph.get(vertex, []):
            if neighbor not in distances:
                distances[neighbor] = distances[vertex] + 1
                queue.append(neighbor)

    return distances
```

## 4. Приклади використання BFS у житті
Навігаційні системи

Алгоритм використовується для пошуку найкоротших маршрутів між пунктами призначення.

Соціальні мережі

Допомагає знаходити друзів друзів та аналізувати зв'язки між користувачами.

Комп'ютерні мережі

Застосовується для пошуку маршрутів передачі даних між вузлами мережі.

Комп'ютерні ігри

Використовується для пошуку шляху персонажа до цілі на карті.

Веб-пошук

Пошукові системи можуть використовувати принципи обходу графів для аналізу вебсторінок і посилань між ними.

---

## Підсумок

BFS — фундаментальний алгоритм обходу графів, що:
- використовує черга (FIFO) для обробки вершин;
- гарантує знаходження найкоротшого шляху в незваженому графі;
- має складність O(V + E);
- широко застосовується там, де потрібен пошук "найближчого" або обхід "рівень за рівнем".
